Meta Tags — Guide

What are meta tags?

Meta tags are head-level elements that provide metadata about the HTML document. They do not display on the page but are used by browsers, crawlers and social platforms.

Important meta tags & attributes

  • charset — sets document character encoding.
  • viewport — controls layout on mobile devices.
  • description — summary shown in search results.
  • robots — control indexing and link following.
  • og:* and twitter:* — social preview metadata.
  • referrer — controls Referer header behavior.
  • theme-color — mobile UI color hint.

SEO best practices (concise)

  1. Write unique, descriptive titles and meta descriptions for every page.
  2. Keep meta description length within ~120–160 characters and avoid keyword stuffing.
  3. Use canonical links to resolve duplicate-content issues.
  4. Provide structured data (JSON-LD) for content types that benefit from rich results.
  5. Use Open Graph and Twitter Card tags to control shared link previews.
  6. Use hreflang for regional/language variants.
  7. Place important meta tags early in the head for predictable rendering.
  8. Test and monitor via Search Console and rich result testing tools.

Common mistakes to avoid

  • Duplicate or missing meta descriptions across many pages.
  • Relying on meta keywords.
  • Hiding critical content behind JavaScript without server-rendered HTML for crawlers.
  • Blocking resources needed for rendering (CSS/JS) via robots.txt.
  • Using meta refresh redirects for permanent moves instead of server-side 301.

Tools & validation

Use Google Search Console, Lighthouse, Mobile-Friendly Test, Rich Results Test and social card validators (Facebook Sharing Debugger, Twitter Card Validator) to validate metadata and previews.
